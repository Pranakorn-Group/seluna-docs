# บล็อกและการจัดการอีเมล

## การจัดการบล็อก

บล็อกช่วยให้ร้านค้าสื่อสารกับลูกค้าผ่านบทความและข่าวสาร

### การดูบทความบล็อก

ไปที่ **แดชบอร์ด → บล็อก** (`/backend/blog`) เพื่อดูบทความทั้งหมด

รายการแสดง:
- ชื่อบทความ
- วันที่สร้าง/เผยแพร่
- สถานะ (ฉบับร่าง / เผยแพร่แล้ว)
- ปุ่มแก้ไขและลบ

### การสร้างบทความใหม่

1. คลิก **สร้างบทความ**
2. กรอกข้อมูล:
   - **ชื่อบทความ** — หัวข้อที่น่าสนใจ
   - **Slug** — URL ที่ใช้เข้าถึงบทความ (ระบบอาจสร้างให้อัตโนมัติจากชื่อ)
   - **บทสรุป (Excerpt)** — คำอธิบายสั้นๆ ที่แสดงในรายการบล็อก
   - **ภาพปก** — อัปโหลดรูปภาพหน้าปกบทความ
   - **เนื้อหา** — เขียนเนื้อหาบทความด้วย rich text editor
3. เลือกสถานะ:
   - **ฉบับร่าง** — บันทึกแต่ยังไม่เผยแพร่ ลูกค้าไม่เห็น
   - **เผยแพร่** — บทความปรากฏบนหน้า `/blog` ของร้านทันที
4. คลิก **บันทึก**

### การแก้ไขบทความ

1. คลิกปุ่ม **แก้ไข** ถัดจากบทความ
2. แก้ไขเนื้อหาหรือข้อมูลที่ต้องการ
3. คลิก **บันทึก**
4. การเปลี่ยนแปลงปรากฏบนหน้าร้านทันที

### การลบบทความ

1. คลิกปุ่ม **ลบ** ถัดจากบทความ
2. ยืนยันการลบ
3. บทความถูกลบออกจากหน้าร้านทันที

## เทมเพลตอีเมล

ระบบส่งอีเมลอัตโนมัติให้ลูกค้าในสถานการณ์ต่างๆ ผู้ดูแลระบบสามารถปรับแต่งเนื้อหาอีเมลเหล่านี้ได้

### การเข้าถึงเทมเพลตอีเมล

ไปที่ **แดชบอร์ด → เทมเพลตอีเมล** (`/backend/email-templates`)

รายการแสดงเทมเพลตที่มีทั้งหมด:

| เทมเพลต | เมื่อไหร่ส่ง |
|---|---|
| ยืนยันออเดอร์ | เมื่อลูกค้าสั่งซื้อสำเร็จ |
| ส่งมอบสินค้าดิจิทัล | เมื่อ credentials ถูกส่งมอบ |
| อัปเดตการจัดส่ง | เมื่อสถานะการจัดส่งเปลี่ยน |
| ตอบกลับออเดอร์บริการ | เมื่อทีมงานตอบกลับ Service Order |
| เติมเงิน Wallet สำเร็จ | เมื่อคำขอเติมเงินได้รับการอนุมัติ |

### การปรับแต่งเทมเพลต

1. คลิกเทมเพลตที่ต้องการแก้ไข
2. แก้ไขเนื้อหาในตัวแก้ไข
3. ใช้ตัวแปรพิเศษสำหรับข้อมูลแบบไดนามิก เช่น:
   - `{{customerName}}` — ชื่อลูกค้า
   - `{{orderId}}` — หมายเลขออเดอร์
   - `{{orderTotal}}` — ยอดรวมออเดอร์
   - `{{trackingNumber}}` — หมายเลขติดตาม
4. คลิก **บันทึก**
5. เทมเพลตที่อัปเดตจะใช้สำหรับอีเมลครั้งต่อไป

## ประวัติอีเมล

ไปที่ **แดชบอร์ด → ประวัติอีเมล** (`/backend/email-logs`) เพื่อดูบันทึกอีเมลทั้งหมดที่ส่งออกไป

รายการแสดง:
- วันที่และเวลาส่ง
- ประเภทอีเมล (ยืนยันออเดอร์ ส่งมอบสินค้า ฯลฯ)
- ผู้รับ (อีเมลลูกค้า)
- สถานะการส่ง (สำเร็จ/ล้มเหลว)
- ออเดอร์ที่เกี่ยวข้อง

### การใช้ประวัติอีเมล

ประวัติอีเมลมีประโยชน์สำหรับ:
- ตรวจสอบว่าลูกค้าได้รับอีเมลยืนยันหรือไม่
- แก้ไขปัญหาเมื่อลูกค้าแจ้งว่าไม่ได้รับอีเมล
- ตรวจสอบการส่งมอบ credentials ให้ลูกค้า

คลิกแต่ละรายการเพื่อดูเนื้อหาอีเมลฉบับเต็มที่ส่งออกไป
